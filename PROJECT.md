## О системе

**Система синтеза речевых объявлений** — профессиональное веб-приложение для генерации речевых объявлений Петербургского метрополитена с соблюдением вещательного стандарта АИ-22.

### Назначение

- Генерация объявлений для громкой связи
- Озвучивание информационных сообщений
- Создание аудиоматериалов для пассажиров

### Ключевые особенности

| Особенность | Описание |
|-------------|----------|
| 🎤 **7 голосов** | Мужские и женские нейронные голоса |
| 🎭 **9 интонаций** | Различные стили произношения |
| 📻 **Стандарт АИ-22** | Broadcast-качество для громкой связи |
| 📱 **PWA** | Устанавливается как приложение |
| 🔐 **Защита** | Парольная аутентификация |

## Стандарт АИ-22

Вещательный стандарт для аудиоконтента на транспорте.

### Параметры

| Параметр | Значение | Описание |
|----------|----------|----------|
| **Формат** | MP3 | Универсальный формат |
| **Частота** | 44100 Гц | CD-качество |
| **Битрейт** | 128 кбит/с CBR | Постоянный битрейт |
| **Каналы** | Моно | Один канал |
| **Кодек** | LAME | Качественный MP3 кодек |
| **LUFS** | -23 ±0.5 LU | Интегрированная громкость |
| **True Peak** | ≤ -1 dBTP | Пиковый уровень |
| **LRA** | ≤ 5 LU | Диапазон громкости |

## API документация

### Аутентификация

#### POST /api/auth
Вход в систему.

```json
// Request
{ "password": "ваш-пароль" }

// Response 200
{ "success": true, "message": "Успешный вход" }

// Response 401
{ "error": "Неверный пароль" }
```

#### GET /api/auth
Проверка сессии.

```json
{ "authenticated": true }
```

#### DELETE /api/auth
Выход из системы.

---

### Генерация аудио

#### POST /api/tts/generate

```json
// Request
{
  "text": "Уважаемые пассажиры! Поезд следует до станции...",
  "voice": "svetlana",
  "rate": "+0%",
  "style": "neutral"
}

// Response
{
  "success": true,
  "audioUrl": "/audio/metro_announcement_xxx.mp3",
  "voice": "Светлана",
  "audioStats": {
    "format": "MP3",
    "sample_rate": 44100,
    "bitrate": "128k CBR",
    "lufs_target": -23.0
  }
}
```

#### GET /api/tts/generate
Список доступных голосов.

### История

#### GET /api/tts/history

#### DELETE /api/tts/history?id={id}

## Установка

### Требования

- Node.js 18+
- Bun
- Python 3.10+
- FFmpeg 4+

### Установка

```bash
# 1. Установить зависимости
bun install

# 2. Создать Python окружение
python3 -m venv tts-env
source tts-env/bin/activate

# 3. Установить Python пакеты
pip install edge-tts pydub numpy pyloudnorm soundfile scipy

# 4. Установить FFmpeg (Ubuntu/Debian)
sudo apt install ffmpeg

# 5. Настроить переменные
cp .env.example .env.local
# Отредактировать .env.local

# 6. Инициализировать БД
bun run db:push

# 7. Запустить TTS сервис (терминал 1)
cd mini-services/tts-service
source ../../tts-env/bin/activate
python3 index.py

# 8. Запустить Next.js (терминал 2)
bun run dev
```

### Переменные окружения

```env
# Пароль для доступа
APP_PASSWORD=ваш-пароль

# Секретный ключ сессии
SESSION_SECRET=случайный-ключ-минимум-32-символа

# URL базы данных
DATABASE_URL=file:./dev.db
```

## Устранение неполадок

### Сервис недоступен

```bash
# Проверить TTS сервис
curl http://localhost:3031/health

# Запустить сервис
cd mini-services/tts-service && python3 index.py
```

### Ошибка пароля

Проверьте файл `.env.local`:
```env
APP_PASSWORD=ваш-пароль
```

### FFmpeg не найден

```bash
# Ubuntu/Debian
sudo apt install ffmpeg

# Проверить
ffmpeg -version
```
